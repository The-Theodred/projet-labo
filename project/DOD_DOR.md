# Definition of Ready (DoR)
- Objectif clair, périmètre défini
- Dépendances identifiées
- Accès/Matériel disponibles
- Critères de test notés

# Definition of Done (DoD)
- Doc à jour (README/runbook)
- Conf dans Git (Ansible ou template)
- Tests passants (ping/service/ports)
- PR revue (1 reviewer min)
- Supervision/Logs visibles si pertinent
- Sauvegarde/restauration testée (si applicable)
